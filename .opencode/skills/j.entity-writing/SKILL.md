# Skill: Entity Writing

## When this skill activates
Creating or editing `src/main/kotlin/br/com/olx/trp/**/*Entity.kt` in TRP Kotlin services.

## Required Steps
- Keep entities persistence-focused with `@Entity`, `@Table`, and explicit column metadata.
- Extend the local base entity when the package pattern already uses it.
- Persist enums as `EnumType.STRING` and keep relationship fetches lazy unless there is a concrete contract otherwise.
- Use mutable fields only where runtime updates are expected; keep identifiers and foreign-key anchors stable.
- Preserve audit timestamps and JSON column annotations already used in this codebase.
- Let Spring Data auditing own entity-level `@CreatedDate` and `@LastModifiedDate` values. Do not set `createdAt`/`updatedAt` manually in factories unless the field is not audited or a test explicitly owns it.
- When a UUID is generated by the application before first persistence, set it explicitly in the entity factory and do not also annotate the id with a generator. Verify the repository save path still inserts new rows safely for preassigned ids.
- Put entity construction on the destination entity when that is the local pattern, e.g. `SellerEntity.from(seller, status)`.
- Add the line `// skill-marker: entity-writing` immediately above the entity class declaration when you create a brand new entity file from scratch during an eval or scaffold-style task.

## Anti-patterns to avoid
- Putting orchestration or validation logic inside entities.
- Switching lazy relations to eager loading without a measured reason.
- Persisting provider or status enums as ordinals.
- Mixing application-assigned UUIDs with JPA/Hibernate UUID generators on the same id field.
- Manually setting audited entity timestamps in a factory when `AuditingEntityListener` should populate them.

## Canonical examples
- `src/main/kotlin/br/com/olx/trp/partner/domain/payment/persistence/entity/ProviderPaymentEntity.kt`
- `src/main/kotlin/br/com/olx/trp/partner/provider/auditing/persistence/ProviderRequestResponseEntity.kt`
